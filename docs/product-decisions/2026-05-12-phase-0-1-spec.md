## 3. Phase 0 — Pre-work deliverables

### 3.1 Palette migration into `ThemeContext`

The existing `ThemeContext` already has a `Theme` interface with ~25 tokens. We do **not** rip it out. We:

1. Replace the `DARK` object's colour values with the new palette.
2. Reduce the existing token surface where it conflicts with the new design rules (e.g. brand-blue accents become the new petrol, hue picker no longer exposed to users).
3. Keep `LIGHT` in place but quietly de-prioritise it. The product is dark-first by intent and the wizard already defaults dark.

**Token mapping — `DARK` object after migration:**

| Token | Old | New | Notes |
|---|---|---|---|
| `bg` | `#0d1117` | `#1A1C1E` | Warm charcoal |
| `dotColor` | `#1b2030` | derive (`#22262a` ~) | Subtle pattern dot |
| `card` | `rgba(255,255,255,0.05)` | `#25282B` (solid) | Per the doc — surface card lift |
| `cardBorder` | `rgba(255,255,255,0.1)` | `rgba(255,255,255,0.06)` | Quieter |
| `cardShadow` | unchanged shape | tone-matched | Soften |
| `text` | `#f3f4f6` | `#E0E0E0` | Off-white |
| `textMuted` | `#9ca3af` | `#A0A4A8` | Tuned to new bg |
| `textFaint` | `#64748b` | `#6B6F73` | Tuned to new bg |
| `btnBg` (primary) | `#f3f4f6` | `#2D5A6E` | Muted petrol — primary CTAs |
| `btnText` | `#111827` | `#E0E0E0` | Off-white on petrol |

### 3.4 Positioning Statement derivation rules (locks Phase 2 input contract)

Phase 2 will use this; we lock the rule now so Phase 1 can quietly persist the field without blocking later work.

**Derivation:**
- **Title:** latest experience entry's `role`, normalised.
- **Seniority signal:** detect "Senior", "Lead", "Principal", "Head", "Manager", "Director" prefix/suffix in title.
- **Years experience:** sum of (endDate − startDate) across all work history entries; cap at 25.
- **Industry/domain:** mode of inferred domain across last 3 employers (use a curated mapping table — banking, tech, government, education, healthcare, retail, etc.). Default "general industry" if unclear.
- **Highest education:** scan `education[].degree` for keywords (PhD > Masters > Bachelor > Diploma > Certificate).

**Output shape (stored on Profile as a single derived field):**

```
positioningStatement: {
  raw: "Senior Financial Analyst with 7 years in mid-market banking, Master's qualified"
  components: {
    title: "Senior Financial Analyst",
    seniority: "senior",
    years: 7,
    domain: "banking",
    education: "masters",
  }
  derivedAt: timestamp
  profileVersion: int
}
```

Re-derive when any source field changes. Expose as read-only on the user's profile in Phase 1, with an edit affordance deferred to v1.1.

### 4.3 Sidebar redesign

**File:** `src/layouts/DashboardLayout.tsx`

**Desktop behaviour:**
- Sidebar collapses to ~64px wide by default (icons only).
- Hovering a nav item shows its label as a small tooltip-style popover anchored to the right edge of the icon.
- A persistent expand affordance (small chevron at the top of the sidebar) toggles a full-width state (240px) and persists in localStorage.
- Logo (`J` mark + "JobReady" text) shows only in expanded state; otherwise just the mark.
- Account/theme/sign-out moves to a single icon-only avatar button at the bottom that opens a small popover menu.

**Touch behaviour (≤ 768px):**
- Sidebar collapses to a single hamburger button at the top-left of the main content area.
- Tapping opens a slide-in drawer from the left, full-height, with full nav labels.
- Tapping outside the drawer or selecting a nav item closes it.
- No hover dependence on touch — the drawer is always full-label.

**Implementation note:** Use a single `<aside>` with conditional rendering driven by:
- `isExpanded` state (desktop expanded/collapsed)
- A media query hook (`useIsTouchOrSmall`) determining touch vs. desktop mode
- Framer Motion for the drawer slide

**Nav items:** unchanged from current set, but re-ordered. Recommended order:
1. Dashboard (`/`) — the Strategy Hub
2. Applications (`/tracker`)
3. Documents (`/documents`)
4. Profile (`/workspace`)
5. Job Feed (`/jobs`) — moves down; feed is a discovery tool, not primary
6. LinkedIn (`/linkedin`)
7. Email Templates (`/email-templates`) — least frequently used, sits last

`/application-workspace` is removed from the visible nav. It's reached *only* via the analysis flow once Phase 2 ships. Direct URL access still works but no nav surface.

### 4.6 `StrategicInsightsPanel.tsx` — component contract

**Content source:** curated `INSIGHTS` array exported from `src/data/strategicInsights.ts` (new file).

**Format:**
```
{
  id: string,
  text: string,           // The insight itself
  category: "market" | "framing" | "process",
  australiaSpecific: boolean,
}
```

**Initial set (12 insights for v1):**

1. Target APAC-focused firms — they value international perspective and often have built-in cultural diversity.
2. Many ASX-listed companies hire through external recruiters — apply via the recruiter URL, not the company site.
3. "Group of Eight" framing helps: e.g. "IIT Delhi (comparable to Group of Eight)" makes your education legible.
4. Government roles (APS) reward selection-criteria detail; treat them as a different application format entirely.
5. Australian cover letters tend to be shorter and more direct than US ones — one page, no flourish.
6. PR/citizenship requirements are common in government roles — check this before tailoring.
7. Industries growing in Australia for international graduates: renewables, aged care, infrastructure, fintech.
8. Melbourne and Sydney differ on tone — Melbourne reads more collegial, Sydney more direct. Calibrate cover letters.
9. Recruiters in AU often respond to thoughtful follow-up within 7 days of application.
10. Avoid US-style superlatives ("rockstar", "ninja", "10x") — they read as imported jargon.
11. AU resumes can include a brief personal-interests line — it's not unprofessional here.
12. Mention your visa status proactively on PR/citizenship-restricted roles; silence reads as a flag.

**Display behaviour:**
- Show 3 insights at a time.
- Rotate on each dashboard mount (random sample without replacement until pool exhausted, then reset).
- Each insight is a small, quiet card — no icons, no emojis, no exclamation marks.
- Section heading: *"Insights for Australian job hunts"*.

**Explicit non-features in v1:**
- No personalisation (the insights don't filter by user role/industry yet).
- No "Was this helpful?" feedback widget.
- No LLM generation.
- No fabricated statistics. Every insight must be true or directional; flag any that aren't during review.

### 4.9 Dimming primitive

**New file:** `src/components/Dim.tsx`

A reusable wrapper that:
- Wraps a parent container
- Tracks a `focusedSelector` (e.g. `"[data-focus-target]"`)
- When a child element matching that selector has focus OR contains the active text input, all other children get `opacity: 0.5` with a 200ms transition
- Clicking outside the focused element restores normal opacity
- Never sets `pointer-events: none` — dimmed elements remain interactive

**Usage in Phase 1:** The `AnalysisHeroCard` sets `data-focus-target`. When the user focuses the textarea, `StrategicInsightsPanel` and `PipelineGlance` dim. Header stays full-strength.

**Reused in Phase 3** for the workspace editors. Built once, used in 4–5 places.
