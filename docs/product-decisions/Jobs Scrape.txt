  Scraping Infrastructure Overview

  1. Firecrawl Service (server/src/services/ingestion/firecrawl.ts)

  - Purpose: Primary scraping engine using Firecrawl API
  - Endpoint: https://api.firecrawl.dev/v1/scrape
  - Features:
    - Bot challenge detection (Cloudflare, captcha, etc.)
    - Job listing validation
    - 30s timeout, 2 retries
    - Mobile user agent for better success
    - 2s wait for JavaScript rendering
    - fetchJobDescription() - dedicated function for fetching full job descriptions

  8. Data Flow

  User requests feed
      ↓
  job-feed.ts (route)
      ↓
  scrapeJobsForTitles() → runIngestionForTitle()
      ↓
      ├─→ Cache check (prisma.job)
      ├─→ Firecrawl adapters (SEEK, Indeed, Jora)
      └─→ Adzuna API
      ↓
  mergeSources() → persistMergedJobs()
      ↓
  prisma.job (persistent cache)
      ↓
  prisma.jobFeedItem (user's daily feed)

  ---
  10. Migration Hygiene (Critical)

  Always write idempotent migrations for staging/production:
  - CREATE TABLE IF NOT EXISTS (not CREATE TABLE)
  - CREATE INDEX IF NOT EXISTS (not CREATE INDEX)
  - ALTER TABLE ... ADD COLUMN IF NOT EXISTS
  Never use `db push` on staging/production; always use `prisma migrate deploy`.
  This prevents P3009/P3018 crash-loops when tables already exist from prior `db push`.
