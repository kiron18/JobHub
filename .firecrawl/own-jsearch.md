## FAQ

Most common questions and answers

### What is the JSearch Jobs API used for?

* * *

JSearch is a real-time jobs API for building job boards, career platforms, HR tools, job market analysis dashboards, and recruiting systems. It returns live job listings, employer profiles, salary estimates, and apply links pulled from Google for Jobs and the open web (covering LinkedIn, Indeed, Glassdoor, ZipRecruiter, and many other publishers).

### How do I search for jobs in countries outside the US?

* * *

Use the country parameter (ISO code) and language parameter together, and include the location in the query. Example for Germany: query=developer in berlin&country=de&language=de. Example for Argentina: query=empleos de desarrollador en buenos aires&country=ar&language=es. Supported country codes include dz, ao, ar, at, br, ca, cl, co, de, eg, fr, gh, gr, hk, id, in, iq, it, jp, ke, mx, my, ng, nl, om, pa, ph, pk, pt, qa, ru, sa, sg, so, es, lk, ch, tw, th, ae, uk, gb, us, vn, za, and many more.

### Why am I getting fewer results than expected for my query?

* * *

The most common fixes: (1) Always include a location in your query, e.g. "software engineer in new york" rather than just "software engineer". (2) For non-US queries, set the correct country and language parameters. (3) If using date\_posted=today or date\_posted=3days, try a wider range. Google for Jobs may have fewer freshly indexed jobs for that query.

### How do I filter results to a specific publisher like LinkedIn, Indeed, or Glassdoor?

* * *

Add "via <publisher>" to your query string. Examples: "developer in new york via linkedin", "marketing manager in sf via indeed", "developer in new york via glassdoor". This works for most major publishers.

### How do I get only remote jobs?

* * *

Set work\_from\_home=true to return only remote / work-from-home jobs. The default is false (or omitting the parameter), which returns both remote and onsite jobs.

### Why are job\_min\_salary, job\_max\_salary, and job\_salary\_period returning null?

* * *

Salary fields are only populated when the employer explicitly includes salary data in the job posting. Most postings do not include salary data, so null values are expected. For salary estimates per job title and location, use the /estimated-salary endpoint (also included with JSearch).

### How do I paginate through job search results?

* * *

Use the /search-v2 endpoint with the cursor parameter. Each response returns a cursor value that you pass back on the next request to fetch the next page. The original /search endpoint had limited pagination depth, so /search-v2 with cursor pagination is the recommended path for any workflow that needs more than the first page of results.

### Can I get a custom plan with higher limits or a hard cap?

* * *

Yes. Contact us at support@openwebninja.com with your expected monthly volume and any requirements (hard request caps to prevent overage charges from automated retries, higher RPS). Custom plans can combine custom request limits, rate limits, and per-request pricing tailored to your usage.

### How do I subscribe to and start using the JSearch API?

* * *

Click the "Try for Free" button at the top of this page to sign up on the OpenWeb Ninja developer portal, subscribe to a plan, and start calling the API by passing your key in the x-api-key header. No credit card required for the free tier. The API is also available on the RapidAPI marketplace if that fits your workflow better (link available at the top). Both routes serve identical endpoints, parameters, and response shapes; the direct portal route avoids RapidAPI's data transfer limits.
